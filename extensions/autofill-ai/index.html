AutoFill AI Privacy Policy

Effective Date: July 1, 2026

1. Overview

AutoFill AI is a Chrome extension that helps users fill web forms by extracting structured data from text they provide. The user pastes or reads clipboard content (such as a business card, resume, or address block), and the extension extracts structured information (name, email, phone, address, etc.) and automatically fills matching form fields on the current webpage. By default, all text parsing is performed locally in your browser using a built-in parser that supports JSON, key-value pairs, and auto-detection of phone numbers, emails, URLs, and ID cards — no AI API key or network connection required. Optionally, if you configure an AI provider, the extension can use AI for enhanced natural language extraction. The extension also includes a universal dropdown detection algorithm that works with any component library, a smart local caching system to speed up repeated operations, and an AI semantic matching fallback that resolves field mapping failures when local synonym matching cannot find a match. All standard data processing happens locally in your browser.

2. Data Handled by the Extension

AutoFill AI handles the following data to provide its core functionality:

  • User-provided text: Text that you paste or read from clipboard is parsed locally by default using the built-in parser (JSON, key-value pairs, regex auto-detection). If you have configured an AI provider and local parsing is insufficient, the text is sent to your configured AI provider for enhanced structured data extraction.
  • Web page form field attributes: After the user activates the extension, the extension scans visible form fields on the current page and reads their attributes (name, id, placeholder, aria-label, label text) to match extracted data to the correct fields. When local synonym-based matching fails for certain data keys, the unmatched keys and the remaining form field labels are sent to the AI provider for semantic analysis to determine the correct field mapping (only if AI is configured). The result is shown to the user with a clear indicator that the mapping was AI-guessed.
  • Settings: Selected AI provider, AI model name, custom endpoint URL, and related configuration.
  • API credentials: API keys entered by the user for OpenAI, Anthropic, or a custom OpenAI-compatible endpoint (optional — not required for basic usage).
  • Text extraction cache: Both local parser results and AI extraction results are cached locally, keyed by a hash of the input text (first 500 characters), with a 24-hour expiry. This avoids repeated parsing or AI calls for identical text. The cache stores only the extracted key-value pairs (e.g., {name: "John", phone: "+1-555-0123"}) — not the original raw text.
  • Form mapping cache: Form structure fingerprints (a hash of field count, field labels, and field types) are cached locally per URL path, with a 7-day expiry. When the same form structure is encountered again, the extension can skip the slower form-aware AI analysis. The fingerprint contains no raw page content — only a numeric hash and field count.

3. Local Processing

Text parsing, form field scanning, synonym-based field matching, dropdown option matching, value injection, and DOM event triggering happen entirely within your browser. AutoFill AI does not operate a server and does not collect, store, or transmit any user data to developer-controlled servers.

  • Text parsing (JSON, key-value pairs, phone/email/URL/ID card auto-detection) runs locally with no network requests.
  • Webpage form fields are scanned only after you click the extension and trigger extraction on the active tab.
  • Field matching primarily uses local synonym mapping and attribute comparison. When local matching fails and AI is configured, an AI call is made to semantically analyze the unmatched keys against form field labels — this only sends the unmatched data keys and field labels, not the full page content.
  • Cache data (extraction results and form fingerprints) is stored and used entirely in the browser.
  • No analytics, telemetry, or usage tracking is included.

4. Local Storage

Settings, API credentials, text extraction cache, and form mapping cache are stored locally using chrome.storage.local. AutoFill AI does not operate a server and does not collect this data on a developer-controlled server. All cache data auto-expires (text cache: 24 hours, form cache: 7 days) and can be manually cleared from the extension settings page at any time.

5. AI Features (Optional)

AutoFill AI works without any AI configuration — the built-in local parser handles JSON, key-value pairs, and auto-detects phone numbers, emails, URLs, and ID cards. However, if you choose to configure an AI provider for enhanced natural language extraction, the extension sends the following data directly from the browser to the AI provider selected by you:

  • The text you pasted or read from clipboard
  • Form field labels and attributes (when form-aware extraction is used, to help the AI match data to specific fields)

Additionally, when local field matching cannot resolve certain extracted data keys, a second AI call is made for semantic matching. This call sends only:

  • The unmatched data keys and their values
  • The remaining form field labels and attributes

The AI suggests which form field each unmatched key likely corresponds to. The result is clearly marked in the UI as an AI-guessed mapping so you are always aware of which fields were filled by inference.

Supported providers:

  • OpenAI (api.openai.com), if the user chooses OpenAI
  • Anthropic (api.anthropic.com), if the user chooses Anthropic
  • A custom OpenAI-compatible endpoint configured by the user (e.g. local Ollama or a remote server)

API credentials are sent only to the selected provider endpoint as required for authentication. The selected provider processes requests under its own terms and privacy policy. No data passes through any intermediate server operated by the extension developer.

6. Data Sharing

AutoFill AI does not sell user data, does not use user data for advertising, and does not share user data with data brokers or advertising networks. By default, all text parsing happens locally in your browser with no data transmitted externally. If you configure an AI provider, text you paste and form field attributes are transferred only as necessary to provide the enhanced extraction feature through the selected AI provider. No data passes through any intermediate server operated by the extension developer.

7. Analytics and Tracking

AutoFill AI does not include analytics SDKs, tracking cookies, advertising code, affiliate code, or behavioral tracking.

8. Chrome Permissions

  • storage: Saves settings, API credentials, text extraction cache, and form mapping cache locally.
  • activeTab: Accesses the current webpage's form fields only after you click the extension and trigger filling.
  • scripting: Injects the content script into the current webpage when the content script was not automatically loaded (e.g., pages opened before the extension was installed).
  • clipboardRead: Reads clipboard text when you click the "Read Clipboard" button in the popup.
  • Host permissions for https://api.openai.com/* and https://api.anthropic.com/*: Sends AI extraction requests to built-in AI endpoints.
  • Optional host permissions for custom http:// and https:// endpoints: Requested only when you configure a custom remote endpoint.

9. Limited Use Statement

The use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. AutoFill AI uses user data only to provide its single purpose: automated form filling from user-provided text in the browser.

10. Children's Privacy

AutoFill AI is a general-purpose form filling tool and is not directed to children under 13. AutoFill AI does not knowingly collect personal information from children.

11. Changes to This Policy

This policy may be updated when AutoFill AI changes its data handling practices. The current policy should be made available from the Chrome Web Store listing and the extension settings page.

12. Contact
