Scraper Privacy Policy

Effective Date: June 20, 2026

1. Overview

Scraper is a Chrome extension that extracts structured data from webpages. Users select elements by clicking on them; the extension auto-detects similar list items, previews results in a side panel, and exports data as CSV or Excel files. An optional AI mode generates extraction selectors from a natural-language description using the user's own API key.

2. Data Handled by the Extension

Scraper handles the following data to provide its core functionality:

  • Webpage DOM content: After the user activates the extension, visible page content is read to detect selectable elements, recognise list structures, and extract text from clicked elements.
  • Settings: Selected AI provider, AI model name, custom endpoint URL, and related configuration.
  • API credentials: API keys entered by the user for OpenAI, Anthropic, or a custom OpenAI-compatible endpoint.
  • Screenshot (optional): Users may paste or upload a single screenshot (PNG / JPEG / WebP, max 4 MB) to help the AI better understand page layout. The screenshot is held only in browser memory for the current session and is never written to persistent storage.

3. Local Storage

Settings, API credentials, and lightweight UI state are stored locally using chrome.storage.local. Scraper does not operate a server and does not collect this data on a developer-controlled server. Screenshots are never stored — they exist only in the active browser session's memory and are discarded when the toolbar is closed or the page is refreshed.

4. Third-Party AI Services

When the user enables optional AI extraction, Scraper sends the following data directly from the browser to the AI provider selected by the user:

  • HTML snippets of the active webpage
  • Page metadata (URL, hostname, title, meta description)
  • The user's natural-language extraction description
  • Optionally, a screenshot the user has manually pasted or uploaded

Supported providers:

  • OpenAI (api.openai.com), if the user chooses OpenAI
  • Anthropic (api.anthropic.com), if the user chooses Anthropic
  • A custom OpenAI-compatible endpoint configured by the user (e.g. local Ollama or a remote server)

API credentials are sent only to the selected provider endpoint as required for authentication. The selected provider processes requests under its own terms and privacy policy. If the user provides a screenshot, it is sent to the same provider as an image attachment and is never stored or forwarded to any developer-operated server.

5. Data Sharing

Scraper does not sell user data, does not use user data for advertising, and does not share user data with data brokers or advertising networks. Webpage content, API credentials, and screenshots are transferred only as necessary to provide the user-requested extraction feature through the selected AI provider. No data passes through any intermediate server operated by the extension developer.

6. Analytics and Tracking

Scraper does not include analytics SDKs, tracking cookies, advertising code, affiliate code, or behavioral tracking. No usage data or scraping activity is reported to any external service.

7. Chrome Permissions

  • activeTab: Accesses the current webpage's DOM only after the user clicks the extension popup and starts a scraping session.
  • scripting: Programmatically injects the local content script into the active tab only after user activation.
  • storage: Stores settings, API keys, and lightweight UI state locally on the user's device.
  • sidePanel: Opens Chrome's side panel to display the data preview table.
  • Host permissions for https://api.openai.com/* and https://api.anthropic.com/*: Used only when the user enables AI extraction with the respective provider.
  • Host permissions for http://localhost/*, https://localhost/*, http://127.0.0.1/*, and https://127.0.0.1/*: Used when the user configures a local AI endpoint.
  • Optional host permissions for custom http:// and https:// endpoints: Requested only when the user configures a custom remote endpoint at a domain not covered above.
  • Host access to http://*/* and https://*/* through the content script: Reads page content and injects visual overlays for element selection.

8. Limited Use Statement

The use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. Scraper uses user data only to provide and improve its single purpose: structured web data extraction in the browser.

9. Children's Privacy

Scraper is a general-purpose data extraction tool and is not directed to children under 13. Scraper does not knowingly collect personal information from children.

10. Changes to This Policy

This policy may be updated when Scraper changes its data handling practices. The current policy is available from the Chrome Web Store listing and the extension's privacy page.

11. Contact

For privacy questions, use the developer contact email listed on the Chrome Web Store listing.
