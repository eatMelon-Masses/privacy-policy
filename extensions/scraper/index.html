Scraper Privacy Policy

Effective Date: June 18, 2026

1. Overview

Scraper is a Chrome extension for web data extraction. It allows you to hover, click, and select data on any webpage, automatically detects list structures, and exports data to CSV or Excel files. All standard data processing happens locally in your browser.

2. Data Handled by the Extension

Scraper handles the following data to provide its core functionality:

  • Web page DOM data: Elements selected by the user on the current page are read from the DOM for extraction.
  • Settings: AI provider configuration, model selection, and display preferences.
  • API credentials: API keys entered by the user for OpenAI, Anthropic, or a custom OpenAI-compatible endpoint (used only for AI-powered extraction).

3. Local Processing

Standard data extraction, list recognition, field matching, and file generation (CSV / Excel) happen entirely within your browser. Scraper does not operate a server and does not collect, store, or transmit any user data to developer-controlled servers.

  • Webpage data is read from the DOM only after you click the extension and start scraping on the active tab.
  • CSV and Excel files are generated locally using browser-side JavaScript.
  • No analytics, telemetry, or usage tracking is included.

4. Local Storage

Settings and API credentials are stored locally using chrome.storage.local. Scraper does not operate a server and does not collect this data on a developer-controlled server.

5. AI Features (Optional)

When you choose to use the optional AI-powered extraction, Scraper sends webpage HTML snippets directly to the AI provider selected by you:

  • OpenAI (api.openai.com), if you choose OpenAI
  • Anthropic (api.anthropic.com), if you choose Anthropic
  • A custom OpenAI-compatible endpoint configured by you, if you choose Custom

API credentials are sent only to the selected provider endpoint as required for authentication. No data passes through any intermediate server. If you configure a remote custom endpoint, Scraper requests permission to access that endpoint's origin. The selected provider processes requests under its own terms and privacy policy.

6. Data Sharing

Scraper does not sell user data, does not use user data for advertising, and does not share user data with data brokers or advertising networks. Scraper transfers webpage snippets and API credentials only as necessary to provide the user-requested AI extraction feature through the selected AI provider.

7. Analytics and Tracking

Scraper does not include analytics SDKs, tracking cookies, advertising code, affiliate code, or behavioral tracking.

8. Chrome Permissions

  • storage: Saves settings and API credentials locally.
  • activeTab and scripting: Injects the scraper into the current webpage only after you click the extension to start scraping.
  • Host permissions for https://api.openai.com/*, https://api.anthropic.com/*, http://localhost/*, https://localhost/*, http://127.0.0.1/*, https://127.0.0.1/*: Sends AI extraction requests to built-in and local AI endpoints.
  • Optional host permissions for custom http:// and https:// endpoints: Requested only when you configure a custom remote endpoint.

9. Limited Use Statement

The use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. Scraper uses user data only to provide its single purpose: web data extraction and export in the browser.

10. Children's Privacy

Scraper is a general-purpose data extraction tool and is not directed to children under 13. Scraper does not knowingly collect personal information from children.

11. Changes to This Policy

This policy may be updated when Scraper changes its data handling practices. The current policy should be made available from the Chrome Web Store listing and the extension settings page.

12. Contact

For privacy questions, use the developer contact email listed on the Chrome Web Store listing.
