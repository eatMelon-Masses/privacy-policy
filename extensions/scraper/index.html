Scraper Privacy Policy

Effective Date: June 24, 2026

1. Overview

Scraper is a Chrome extension for web data extraction. It allows you to hover, click, and select data on any webpage, automatically detects list structures, and exports data to CSV or Excel files. The extension includes built-in presets for 20+ popular websites, a smart scroll system for dynamic content, and a network response analyzer that reads page API responses for richer structured data. All standard data processing happens locally in your browser.

2. Data Handled by the Extension

Scraper handles the following data to provide its core functionality:

  • Web page DOM data: After the user activates the extension, visible page content is read to detect selectable elements, recognise list structures, and extract text from clicked elements.
  • Network API responses: When the scraper is activated, fetch and XHR responses from the current page are intercepted in browser memory for local structured data analysis. This data never leaves your browser.
  • Settings: Selected AI provider, AI model name, custom endpoint URL, and related configuration.
  • API credentials: API keys entered by the user for OpenAI, Anthropic, or a custom OpenAI-compatible endpoint.
  • Cached extraction plans: Successful extraction patterns (CSS selectors and field names) are cached locally per hostname to speed up re-extraction. No page content or user data is stored in the cache.
  • Screenshot (optional): Users may paste or upload a single screenshot (PNG / JPEG / WebP, max 4 MB) to help the AI better understand page layout. The screenshot is held only in browser memory for the current session and is never written to persistent storage.

3. Local Processing

Standard data extraction, list recognition, field matching, network response analysis, and file generation (CSV / Excel) happen entirely within your browser. Scraper does not operate a server and does not collect, store, or transmit any user data to developer-controlled servers.

  • Webpage data is read from the DOM only after you click the extension and start scraping on the active tab.
  • Network API responses (fetch/XHR) are intercepted in browser memory for structured data analysis and never transmitted externally.
  • CSV and Excel files are generated locally using browser-side JavaScript.
  • No analytics, telemetry, or usage tracking is included.

4. Local Storage

Settings, API credentials, cached extraction plans (site pattern memory), and lightweight UI state are stored locally using chrome.storage.local. Scraper does not operate a server and does not collect this data on a developer-controlled server. Screenshots are never stored — they exist only in the active browser session's memory and are discarded when the toolbar is closed or the page is refreshed.

5. AI Features (Optional)

When you choose to use the optional AI-powered extraction, Scraper sends the following data directly from the browser to the AI provider selected by you:

  • HTML snippets of the active webpage
  • Page metadata (URL, hostname, title, meta description)
  • The user's natural-language extraction description
  • Optionally, a screenshot the user has manually pasted or uploaded

Supported providers:

  • OpenAI (api.openai.com), if the user chooses OpenAI
  • Anthropic (api.anthropic.com), if the user chooses Anthropic
  • A custom OpenAI-compatible endpoint configured by the user (e.g. local Ollama or a remote server)

API credentials are sent only to the selected provider endpoint as required for authentication. The selected provider processes requests under its own terms and privacy policy. No data passes through any intermediate server operated by the extension developer. If the user provides a screenshot, it is sent to the same provider as an image attachment and is never stored or forwarded to any developer-operated server.

6. Data Sharing

Scraper does not sell user data, does not use user data for advertising, and does not share user data with data brokers or advertising networks. Webpage content, API credentials, and screenshots are transferred only as necessary to provide the user-requested extraction feature through the selected AI provider. No data passes through any intermediate server operated by the extension developer.

7. Analytics and Tracking

Scraper does not include analytics SDKs, tracking cookies, advertising code, affiliate code, or behavioral tracking.

8. Chrome Permissions

  • storage: Saves settings, API credentials, and cached extraction plans locally.
  • activeTab and scripting: Injects the scraper into the current webpage only after you click the extension to start scraping.
  • sidePanel: Opens Chrome's side panel to display the data preview table.
  • Host permissions for https://api.openai.com/*, https://api.anthropic.com/*, http://localhost/*, https://localhost/*, http://127.0.0.1/*, https://127.0.0.1/*: Sends AI extraction requests to built-in and local AI endpoints.
  • Optional host permissions for custom http:// and https:// endpoints: Requested only when you configure a custom remote endpoint.

9. Limited Use Statement

The use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements. Scraper uses user data only to provide its single purpose: web data extraction and export in the browser.

10. Children's Privacy

Scraper is a general-purpose data extraction tool and is not directed to children under 13. Scraper does not knowingly collect personal information from children.

11. Changes to This Policy

This policy may be updated when Scraper changes its data handling practices. The current policy should be made available from the Chrome Web Store listing and the extension settings page.

12. Contact

For privacy questions, use the developer contact email listed on the Chrome Web Store listing.
